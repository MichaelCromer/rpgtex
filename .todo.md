# RPGTeX — TODO & Low-Hanging Fruit

## 🐛 Bugs

### 1. `rpgheader.sty` missing `\RequirePackage{graphicx}`
Uses `\includegraphics` (lines 41, 51, 62) without loading `graphicx` directly.
Works only because `tikz` (via `rpgcore.sty`) loads it transitively. Fails standalone.

### 2. `rpgfonts.sty` missing color fallback guards
References `titlecolor` and `contourcolor` without providing fallback definitions
when `g__rpg_colors_loaded_bool` is false. Unlike `rpgcomment.sty` / `rpgsidebar.sty`
/ `rpgreadaloud.sty` / `rpgtable.sty`, it is not standalone-safe.

### 3. Unresolved `before~upper` TODO in comment/sidebar/readaloud
All three files have `before~upper = { }, % TODO old code had layout-true ? =>
justified-false ? => \raggedright` — a no-op that needs a decision.
- `rpgcomment.sty:23`, `rpgsidebar.sty:23`, `rpgreadaloud.sty:22`

## 🔧 Larger Improvements (15–30 min each)

### 4. Consolidate duplicated CR/XP data
Challenge Rating → XP mapping appears 2× in `rpgmonster.sty`:
- `\RPG_CRExp` (lines 3–44) — dead code, never called
- `\__rpg_cr_xp:N` (lines 187–240) — actually used at line 334

Extract to a single data structure (e.g. a property list or lookup table).
`\__rpg_cr_pb:N` (lines 245–287) shares the same CR-key structure but maps
to PB values, so it could also benefit from a shared CR-key enumeration.

### 5. Reduce code duplication in comment/sidebar/readaloud
These 3 modules are ~85% structurally identical (~30 boilerplate lines each:
key def + start function + nV variant + env wrapper). A shared base macro
would cut ~12–15 lines per file. The tcolorbox definitions (visual styling)
would remain per-module since they differ significantly (borderlines, overlays,
shadow, boxsep, etc.).

### 6. Set up a build/test system
No `l3build`, no tests, no CI. Only an `examples/Makefile` that compiles
example PDFs with `pdflatex`. A simple `l3build` config or shell script
would be a big quality-of-life improvement.

## 🧹 Cleanup & Modernization

### 7. `rpgmonster.sty` inconsistent indentation
Mixes 2-space and 4-space indentation (670 lines, the largest file).

### 8. `rpgstrings.sty` uses `\newcommand` instead of expl3
All 40+ string definitions use LaTeX2e `\newcommand` rather than expl3
`\tl_const:Nn` or `\str_const:Nn`. Works fine but pollutes the `\` namespace.

### 9. `rpgoptions.clo` — sections marked for removal in 1.0.0
Lines 38–41 and 132–141 are workarounds for the old `layout` boolean
that should be cleaned up before a 1.0 release.

### 10. `rpgheader.sty` — hardcoded image paths
References `img/paper` and `img/footerscroll` with no user override option.

### 11. README is very sparse (3 lines)
No installation instructions, usage examples, API doc, or dependency info.

### 12. No root `.gitignore`
Only `examples/.gitignore` exists; no gitignore for root build artifacts.
