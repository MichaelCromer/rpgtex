# RPGTeX — TODO & Low-Hanging Fruit

## Quick Fixes (5–15 min each)

### 1. Fix duplicate `RPG_Tan3` definition
`rpgcolors.sty:24-25` defines `RPG_Tan3` twice with different values (`{HTML}{F9EFD6}` then `{HTML}{FDF1DC}`). The first is silently overwritten. Likely meant to be `RPG_Tan4` or the duplicate should be removed.

### 2. Wire `rpglayout.sty` into the load chain
`rpglayout.sty` was created in the latest commit but is **not loaded by anything** (`rpgcore.def`, `rpg.sty`, `rpgbook.cls`). Dead code. Needs `\RequirePackage{rpglayout}` in `rpgcore.def`, likely behind the `\l__rpg_layout_bool` conditional.

### 3. Fix `\RPG_FontSubaragraph` typo
`rpgfonts.sty:243` missing `p` — should be `\RPG_FontSubparagraph`. Propagates to `rpglayout.sty:18,112`.

### 4. Fix undefined `\rpg@deprecate` command
`rpgmonster.sty:200` uses `\rpg@deprecate` which is never defined. Codebase has `\__rpg_deprecated:nnn` instead. Replace the call.

### 5. Define missing colors across the codebase
Several files use colors that are **never defined**:
| Color | Used in |
|---|---|
| `titlered` | `rpgfonts.sty`, `rpgmonster.sty`, `rpgreadaloud.sty` |
| `pagegold` | `rpgfonts.sty` |
| `rulered` | `rpgmonster.sty` |
| `statblockbg` | `rpgmonster.sty` |
| `statblockribbon` | `rpgmonster.sty` |
| `contourgray` | likely expected by contour package |

Quickest fix: define them as aliases to existing `RPG_*` colors in `rpgcolors.sty`.

### 6. Add guard-bool fallbacks to comment/readaloud/sidebar
`rpgcomment.sty`, `rpgreadaloud.sty`, and `rpgsidebar.sty` crash when loaded standalone — they reference booleans (`\l__rpg_layout_bool`, `\l__rpg_justified_bool`) and font commands from other files without fallbacks. `rpgtable.sty` already has this pattern. Copy it.

## Larger Improvements (15–30 min each)

### 7. Consolidate duplicated CR/XP data
Challenge Rating → XP mapping appears 3× in `rpgmonster.sty`:
- `\RPG_CRExp` (lines 3–44)
- `\__rpg_cr_xp:N` (lines 187–240)
- `\__rpg_cr_pb:N` (lines 245–287)

Extract to a single data structure (e.g. a property list or lookup table).

### 8. Create an `example.tex` demo file
No example or demo document exists. Creating one that exercises all major environments (monster blocks, comments, sidebars, read-aloud, tables, headers, sections, spells) would make testing and development drastically faster.

### 9. Reduce code duplication in comment/sidebar/readaloud
These 3 modules are structurally nearly identical (~40 lines each: tcolorbox + key def + start function). A shared base macro would reduce maintenance burden.

### 10. Set up a minimal build/test system
No Makefile, no `l3build`, no tests, no CI. A simple `l3build` config or even a shell script that compiles `example.tex` with `latexmk` would be a big quality-of-life improvement.
